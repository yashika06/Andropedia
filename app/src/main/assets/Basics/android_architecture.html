Android Architecture

The Android operating system can be divided into the four areas as depicted in the following graphic. An Android application developer typically works with the two layers on top to create new Android applications.

The levels can be described as:

  • Applications -On the top of android framework, there are applications.All applications such as home, contact, camera, gallery, settings, games, browsers are using android framework that uses android runtime and libraries.

  • Application framework - An API which allows high-level interactions with the Android system. It provides a lot of classes and interfaces for android application development.

  • Libraries and runtime - The libraries for many common framework functions, like graphic rendering, data storage, web browsing. In android runtime, there are core libraries and DVM (Dalvik Virtual Machine) which is responsible to run android application.

  • Linux kernel - Communication layer for the underlying hardware. It is the heart of android architecture that exists at the root of android architecture. Linux kernel is responsible for device drivers, power management, memory management, device management and resource access.
